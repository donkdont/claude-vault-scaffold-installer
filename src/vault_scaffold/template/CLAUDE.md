# CLAUDE.md

This file provides guidance to Claude Code when working with this Obsidian vault.

## Vault Structure

Top-level folders:

| Folder | Purpose |
|---|---|
| `00 Inbox/` | Incoming notes, clippings, sources — use `00 Inbox/Quellen/` for raw material |
| `01 Systeme/` | System and infrastructure documentation |
| `02 Projekte/` | Project notes |
| `03 Wissen/` | Transferable, abstracted knowledge |
| `04 Referenz/` | Concrete reference material (IPs, specs, inventories) |
| `05 Archiv/` | Archived notes — add `archiv` tag, don't delete |
| `06 Forschung/` | Research notes |
| `07 Unternehmen/` | Business and company notes |

## Installed Scripts

All scripts live under `.claude/scripts/`:

| Script | Purpose |
|---|---|
| `update-vault-index.py` | Rebuild `INDEX.md` files from note `description` frontmatter |
| `vault-lint-hook.py` | Auto-check frontmatter and wikilinks after every Write/Edit |
| `queue-description.py` | Add a note to the pending-descriptions queue |
| `check-pending-descriptions.sh` | Report notes waiting for description generation |
| `enforce-index-first.py` | Block reads on non-INDEX files before INDEX.md is read |
| `enforce-sdd-tdd.py` | Block new implementation files without spec/test first |
| `suggest-semantic-search.py` | Suggest semantic search for vault queries |

## Note Frontmatter

Required fields for every note:

```yaml
title: Note Title
tags: [category-tag]
created: YYYY-MM-DD
updated: YYYY-MM-DD
```

Add `code_path` for project/system notes (path to the associated repository).
Category tag = folder name: `system`, `projekt`, `wissen`, `referenz`, `archiv`, `forschung`, `unternehmen`, `quelle`.

Do **not** add `description` manually — it is generated by the vault-describer agent.

## Writing Rules

- Before writing a new note: check if the topic is already covered in an existing note. Extend first, create only if truly new.
- After writing: queue the note for description generation (`queue-description.py`).
- Do not manually edit `INDEX.md` files — they are generated from note descriptions.
- Every new note needs at least one incoming wikilink from an existing note.

## Semantic Search

Use Smart Connections MCP tools for conceptual/exploratory searches:

- `vault_semantic_search(query)` — find notes by topic or concept
- `find_similar(note_path)` — find notes similar to a given note

Requires Obsidian to be running with the Smart Connections plugin active.
